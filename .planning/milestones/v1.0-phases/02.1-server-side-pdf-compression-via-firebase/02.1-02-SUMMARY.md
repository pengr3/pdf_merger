## Decisions Made

- **Size gate at compression time:** The 32 MB limit for server mode is checked in `compressAllFiles()` at the time of compression, not in `addCompressFiles()` at upload time. This allows users to add files and then choose between Fast (80 MB) and Server (32 MB) modes without re-uploading.
- **No Content-Type header:** The `fetch()` call for FormData must NOT set `Content-Type` — the browser sets it automatically with the multipart boundary string. Setting it manually breaks the upload.
- **Privacy notice styling:** Blue info style (not red warning) — sending to server is a feature, not a danger. Red was appropriate for the old Heavy mode which destroyed text.
- **compressPresetValues removed:** The client-side JPEG quality map (`{ best: 0.92, balanced: 0.75, compressed: 0.55 }`) is removed entirely. Preset strings (best/balanced/compressed) are now sent directly to the server, which maps them to Ghostscript `/prepress`, `/ebook`, `/screen` profiles.
