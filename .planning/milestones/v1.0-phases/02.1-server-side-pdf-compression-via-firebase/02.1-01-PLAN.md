Create the Firebase Cloud Function that receives a PDF via multipart POST, compresses it using Ghostscript with user-selected preset, and returns the compressed PDF binary. Set up all Firebase project files and configuration.

**2. `functions/index.js`:**
Create the `compressPdf` Cloud Function exactly as specified in the RESEARCH.md Pattern 2, with these specifications:
- Use `const { onRequest } = require('firebase-functions/v2/https');`
- Configure: `cors: true`, `memory: '512MiB'`, `timeoutSeconds: 300`, `region: 'us-central1'`
- Reject non-POST requests with 405
- Parse multipart using Busboy — read `pdf` file field and `preset` text field (default: `'balanced'`)
- Use Busboy `limits: { fileSize: 50 * 1024 * 1024 }` (50 MB)
- Dual-path for rawBody: `if (req.rawBody) bb.end(req.rawBody); else req.pipe(bb);`
- Validate pdfBuffer is not null/empty — return 400 with `{ error: 'No PDF file received' }`
- Map preset to Ghostscript flag: `{ best: '/prepress', balanced: '/ebook', compressed: '/screen' }`
- Generate unique temp file paths using `crypto.randomUUID()`
- Write input PDF to `/tmp/{uuid}-input.pdf`
- Invoke Ghostscript via `execFile('gs', [...args])` with ALL required flags:
  - `-q` (quiet)
  - `-dNOPAUSE` (no interactive pause)
  - `-dBATCH` (exit after processing)
  - `-dSAFER` (restrict file access)
  - `-sDEVICE=pdfwrite` (PDF output)
  - `-dCompatibilityLevel=1.4` (broad support)
  - `-dPDFSETTINGS=${gsPreset}` (compression preset)
  - `-dEmbedAllFonts=true` (preserve fonts)
  - `-dSubsetFonts=true` (smaller output)
  - `-sOutputFile=${outputPath}` (output path)
  - `inputPath` (input file)
- Read output file and return as `application/pdf` with `Content-Disposition: attachment`
- Always clean up both input and output temp files in `finally` block using `fs.unlink().catch(() => {})`
- Wrap the Ghostscript execution in try/catch — if `execFile` throws, return 500 with `{ error: 'Compression failed' }`

CRITICAL ANTI-PATTERNS (from RESEARCH.md):
- Do NOT use Multer — it is broken in Cloud Functions (body already parsed by middleware)
- Do NOT set Content-Type manually in response — use `res.set()` before `res.send()`
- Do NOT use static temp file names like `/tmp/input.pdf` — concurrent requests will corrupt each other
- Do NOT omit `-dNOPAUSE` or `-dBATCH` — Ghostscript will hang indefinitely
