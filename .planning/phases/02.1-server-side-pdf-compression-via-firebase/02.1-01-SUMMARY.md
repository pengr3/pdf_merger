## Decisions Made

- **Busboy over Multer:** Multer is broken in Cloud Functions (body pre-parsed by middleware). Busboy is the documented workaround per official Cloud Functions docs.
- **Dual-path rawBody:** `if (req.rawBody) bb.end(req.rawBody); else req.pipe(bb)` — handles both production (rawBody always available) and local emulator (may not provide rawBody).
- **Preset mapping:** `best` → `/prepress` (300 DPI), `balanced` → `/ebook` (150 DPI), `compressed` → `/screen` (72 DPI). All three preserve text vectors — only raster images are downsampled.
- **UUID temp files:** `crypto.randomUUID()` generates unique `/tmp/{uuid}-input.pdf` and `/tmp/{uuid}-output.pdf` per request, preventing concurrent request collisions.
- **YOUR_PROJECT_ID placeholder:** `.firebaserc` uses placeholder — user must replace with actual Firebase project ID from Firebase Console before running `firebase deploy`.

## User Setup Required

Task 2 of this plan requires the following manual steps (Firebase cannot be configured programmatically — requires interactive browser authentication and a credit card for Blaze plan):

**Step 1: Create Firebase Project**
1. Go to https://console.firebase.google.com
2. Click "Add project" → enter a project name (e.g., "clmc-pdf-tools")
3. Disable Google Analytics → click "Create project"

**Step 2: Upgrade to Blaze Plan**
1. In the Firebase console, click "Upgrade" in the bottom-left sidebar
2. Select Blaze (pay-as-you-go) plan and add a payment method
3. Note: Free tier is 2M invocations/month — cost is effectively $0 for internal use

**Step 3: Install Firebase CLI and Login**
```bash
npm install -g firebase-tools
firebase login
```

**Step 4: Update .firebaserc**
Edit `.firebaserc` and replace `YOUR_PROJECT_ID` with the actual Firebase project ID (visible in Firebase Console URL or project settings).

**Step 5: Deploy**
```bash
cd C:/Users/franc/dev/pdf_merger
firebase deploy --only functions
```

**Step 6: Note the deployed URL**
After deployment, Firebase prints the function URL:
```
https://us-central1-YOUR_PROJECT_ID.cloudfunctions.net/compressPdf
```
This URL is needed in Plan 02 to configure `window.COMPRESS_PDF_ENDPOINT` in `index.html`.

**Step 7: Verify deployment (optional)**
```bash
curl -X POST "https://us-central1-YOUR_PROJECT_ID.cloudfunctions.net/compressPdf" \
  -F "pdf=@path/to/test.pdf" \
  -F "preset=balanced" \
  -o compressed-output.pdf
```
Open `compressed-output.pdf` and verify text is still selectable.
