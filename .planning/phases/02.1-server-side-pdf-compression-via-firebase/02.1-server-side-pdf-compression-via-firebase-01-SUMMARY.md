# Plan 02.1-01: Firebase Cloud Function Summary

**compressPdf Cloud Function with Ghostscript compression, rate limiting, and tuned presets (Best ~30%, Balanced ~50%, Compressed ~90%)**

## Performance

- **Duration:** ~45 min (including user Firebase setup + DPI tuning iterations)
- **Tasks:** 2 (1 auto + 1 human-action checkpoint)
- **Files created:** 5
- **Files modified:** 1

## Accomplishments
- Cloud Function receives PDF via multipart POST, compresses with Ghostscript, returns binary
- Three presets with explicit DPI and JPEG QFactor control achieving target reduction percentages
- In-memory spike detection: 5 req/min per IP, 30 req/min global — rejects before file parsing
- maxInstances: 2 caps concurrent executions
- Size guard returns original PDF when compression would inflate the file

## Decisions Made
- Busboy over Multer for multipart parsing (Multer broken in Cloud Functions)
- In-memory rate limiting over Firestore counter (simpler, no extra service)
- Dropped -dPDFSETTINGS entirely — it was overriding custom DPI flags
- QFactor via PostScript `-c << >> setdistillerparams -f` syntax
- Iteratively tuned DPI/QFactor to match user's target savings

## Deviations from Plan

### Rate Limiting (user-requested)
- **Issue:** User concerned about abuse/spam on Blaze plan
- **Fix:** Added in-memory sliding window rate limiting + maxInstances: 2
- **Not in original plan** but essential for production safety

### Compression Preset Tuning (user-requested)
- **Issue:** Original Ghostscript presets (/prepress, /ebook, /screen) didn't hit target savings
- **Fix:** Replaced -dPDFSETTINGS with fully explicit DPI + QFactor parameters, tuned iteratively
- **6 tuning commits** to reach target percentages

## Issues Encountered
- `-dPDFSETTINGS` overrides custom DPI flags — resolved by removing it entirely
- PostScript `-c` syntax required `-sOutputFile` before `-c`/`-f` block
- `/prepress` preset inflated already-compressed PDFs — resolved by size guard
